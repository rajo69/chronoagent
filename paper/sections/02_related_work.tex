\section{Related Work}
\label{sec:related}

\subsection{Memory Poisoning Attacks on LLM Agents}
\label{sec:related:attacks}

MINJA~\citep{minja}, AgentPoison~\citep{agentpoison}, and
A-MemGuard~\citep{amemguard} established that shared vector memory is
a viable attack surface for multi-agent LLM systems.
\citet{minja} target the retrieval layer directly by crafting
high-similarity trigger documents; \citet{agentpoison} add a learned
backdoor that activates on a semantic trigger; \citet{amemguard}
propose a defensive retrieval filter but operate per-query without
temporal context.

\subsection{Decentralized Task Allocation in Multi-Agent Systems}
\label{sec:related:alloc}

Classical approaches include contract-net protocols~\citep{contractnet}
and Dec-POMDP solvers~\citep{decpomdp}. More recent work on LLM-based
task allocation, including AutoGen~\citep{autogen},
MetaGPT~\citep{metagpt}, and CrewAI~\citep{crewai}, delegates
coordination to a privileged planner agent; this centralizes trust
and re-introduces the very single-point-of-failure that multi-agent
systems were meant to avoid.

\subsection{Online Change-Point Detection and Forecasting}
\label{sec:related:cpd}

Bayesian Online Change-Point Detection~\citep{adams2007bocpd} provides
a principled run-length posterior with $O(T)$ memory. The Chronos
family of pretrained probabilistic time-series
forecasters~\citep{chronos} gives a zero-shot predictive distribution
we use as a temporal prior on agent behavior; ChronoAgent ships with
the \texttt{amazon/chronos-t5-small} checkpoint (Chronos T5-small, 46M
parameters, Ansari et al. 2024).

\subsection{Runtime Monitoring for LLM Agents}
\label{sec:related:monitoring}

Prior runtime monitors such as NeMo Guardrails~\citep{nemoguardrails},
Llama Guard~\citep{llamaguard}, and constitutional-AI
classifiers~\citep{constitutionalai} focus on content-level filtering
at the input and output boundary. They are complementary to \sys{}:
they answer ``is this specific message safe?'' while \sys{} answers
``has this agent's behavior drifted over the last $k$ steps?''.
